# ARCHITECTURE_LIQPAY.md

> Архівний документ раннього дизайну інтеграції LiqPay.

## Статус

Цей файл описує **попередній архітектурний план**, у якому LiqPay інтегрувався через окремий generic provider layer (`payment_providers`).

Поточна реалізація в репозиторії вже спрощена:
- LiqPay налаштовується **на рівні отримувача** (`receivers`)
- активний runtime flow більше не покладається на окрему адмінку generic providers
- частина старих DDL/нотаток може лишатися лише для історичного контексту або backward compatibility

## Де дивитись актуальний стан

Канонічні джерела зараз:
- `README.md`
- `ARCHITECTURE.md`
- `app.py`
- `db.py`
- `schema.sql`

## Навіщо збережено файл

- як історія еволюції дизайну
- як контекст для майбутніх рефакторингів
- щоб було видно, чому в схемі БД можуть траплятися артефакти ранньої моделі

Якщо generic provider layer більше не потрібен зовсім, цей файл у майбутньому можна видалити разом із залишками відповідного DDL.
